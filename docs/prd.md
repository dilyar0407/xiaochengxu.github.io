# 体育课预约系统需求文档

## 1. 应用概述

### 1.1 应用名称
体育课预约系统

### 1.2 应用描述
一款基于微信小程序的体育课预约管理系统，为用户提供便捷的课程浏览、在线预约、记录查询等功能，支持微信授权登录，实现前端与后端的完整交互。\n
## 2. 核心功能

### 2.1 课程列表展示
- 按运动类型分类展示可预约课程（如篮球、羽毛球、瑜伽、游泳等）\n- 课程信息包含：课程名称、上课时间、上课地点、剩余名额、授课老师
- 支持按运动类型筛选课程
- 支持下拉刷新刷新课程列表

### 2.2 预约功能
- 用户选择课程后提交预约申请
- 系统校验剩余名额，名额充足方可预约
- 预约成功后给出明确提示
- 预约失败时说明原因（如名额已满、预约截止等）\n- 加载中显示加载动画

### 2.3 预约记录管理
- 查看个人预约记录，记录状态包括：已预约、已完成、已取消
- 支持取消未开始课程的预约
- 超过预约截止时间不可取消
- 取消后名额自动释放

### 2.4 用户登录与个人中心
- 基于微信授权登录，获取用户昵称和头像
- 自动生成用户唯一标识
- 个人中心展示：用户头像、昵称、我的预约记录、预约规则说明

## 3. 页面结构

### 3.1 首页（课程列表页）\n- 顶部导航栏：包含搜索框、分类筛选入口
- 课程卡片列表：展示课程基本信息及状态
- 底部tabbar：课程列表、我的预约

### 3.2 课程详情页
- 课程详细信息展示
- 预约按钮（可预约/已预约/已满员状态）\n
### 3.3 我的预约页
- 预约记录列表，按状态分类展示
- 每条记录显示课程信息、状态、操作按钮（取消预约/查看详情）\n
### 3.4 个人中心页
- 用户基本信息展示
- 预约记录入口
- 预约规则说明

## 4. 技术要求

### 4.1 前端开发
- 基于微信小程序原生语法（WXML/WXSS/JS）开发
- 适配不同手机屏幕尺寸，样式简洁清晰
- 封装请求工具类，统一处理接口请求/响应、异常情况
- 实现下拉刷新、加载动画等交互效果

### 4.2 后端接口
- 课程列表查询接口
- 预约提交接口
- 预约记录查询接口
- 取消预约接口
- 接口返回格式统一，支持参数校验
- 支持跨域配置，适配小程序请求规则

### 4.3 运行要求
- 确保在微信开发者工具中编译、预览、真机调试均无报错
- 控制台无红/黄报错信息
- 前后端联调正常，功能完整
