
项目介绍
本项目是一款面向学生群体的体育课预约微信小程序，旨在解决传统体育课选课流程繁琐、场地资源分配不均、预约信息不透明等问题。通过小程序，学生可便捷查看可预约的体育课程、场地及时间段，在线完成预约与取消操作；管理员可后台管理课程信息、场地资源及用户预约记录，实现体育教学资源的高效统筹与管理。

功能特性

1. 学生端功能

- 用户登录：支持微信快捷登录，自动关联学生身份信息（需后台提前录入学生数据）。

- 课程查看：按体育项目（如篮球、足球、羽毛球等）、上课时间段、场地位置分类展示可预约课程，清晰呈现课程剩余名额。

- 在线预约：选择心仪课程后，确认预约信息并提交，预约成功后生成预约凭证（含课程详情、预约时间、场地位置等）。

- 预约管理：查看个人历史预约记录，支持在预约截止前取消预约，释放课程名额。

- 消息通知：预约成功、预约即将开始、课程调整/取消等关键信息通过微信服务通知推送。

- 个人中心：查看个人资料、已预约课程统计，修改个人信息（如联系方式）。

2. 管理员端功能

- 管理员登录：独立管理员账号密码登录，区分普通管理员与超级管理员权限。

- 课程管理：新增、编辑、删除体育课程信息（含课程名称、体育项目、上课教师、上课时间、场地、名额限制等）。

- 场地管理：维护体育场地信息（如场地名称、位置、可开展项目、容量、开放时间等），设置场地占用状态。

- 预约审核（可选）：针对特殊课程（如专业训练课）可开启预约审核，审核通过后预约生效。

- 数据统计：查看课程预约率、场地使用率、热门体育项目排行等数据，生成可视化报表。

- 用户管理：查看学生用户列表，维护学生身份信息（新增、编辑、禁用等）。

- 系统公告：发布课程调整、场地维护、活动通知等公告，推送给所有学生用户。

环境要求

1. 开发环境

- 微信开发者工具：稳定版 1.06.2409090 及以上

- 基础库版本：2.30.0 及以上

- 开发语言：WXML、WXSS、JavaScript/TypeScript、Node.js（后端接口）

- 数据库：MySQL 8.0 及以上（存储用户、课程、预约等数据）

- 服务器：支持 Node.js 运行环境，推荐阿里云、腾讯云等云服务器

2. 运行环境

- 微信客户端：iOS 8.0 及以上、Android 6.0 及以上

- 小程序已完成备案及微信开放平台接入（获取 AppID）

- 服务器配置：2核4G 及以上，带宽 2M 及以上（支持并发访问）

项目部署流程

1. 前期准备

1. 登录微信公众平台，注册小程序账号，完成实名认证，获取小程序 AppID。

2. 配置服务器环境：安装 Node.js、MySQL，搭建后端项目运行环境（推荐使用 Express/Koa 框架）。

3. 导入数据库脚本：在 MySQL 中创建项目数据库，执行项目根目录下的 sql/pe_reservation.sql 脚本，初始化数据表结构及测试数据。

2. 后端项目部署

1. 将后端项目代码上传至服务器，进入项目目录执行 npm install 安装依赖包。

2. 修改后端配置文件（config/config.js）：配置数据库连接信息（主机、端口、用户名、密码、数据库名）、小程序 AppID、AppSecret 等。

3. 使用 PM2 启动后端服务：执行 pm2 start app.js --name pe-reservation-backend，确保服务稳定运行。

4. 配置服务器安全组：开放 3000 端口（或自定义后端服务端口），允许外部访问。

3. 前端项目配置与预览

1. 下载并安装 微信开发者工具，打开前端项目代码。

2. 在微信开发者工具中配置项目：填写项目名称、项目目录，输入已获取的小程序 AppID，勾选“不校验合法域名”（开发环境）。

3. 修改前端配置文件（utils/config.js）：将后端接口基础地址（baseUrl）修改为部署后的服务器地址（如 http://xxx.xxx.xxx.xxx:3000）。

4. 点击“编译”按钮，在模拟器中预览小程序功能，调试接口调用、页面交互等是否正常。

4. 正式发布

1. 在微信开发者工具中，点击“上传”按钮，填写版本号、更新说明，将前端代码上传至微信公众平台。

2. 登录微信公众平台，进入“版本管理”，找到上传的小程序版本，提交审核。

3. 审核通过后，在“版本管理”中发布小程序，用户即可在微信中搜索小程序名称使用。
